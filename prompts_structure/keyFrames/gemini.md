# KeyFrames · Gemini 9-Grid Keyframe Reference

## What It Is

KeyFrames is a single **3×3 full-color keyframe grid** generated by Gemini in one pass, covering every shot in a scene. Each cell is the definitive visual lock for that shot — color, lighting, composition, and character appearance are all finalized. The video model no longer receives open-ended "concept art + text description" input — it receives a deterministic "this is exactly what this shot looks like" reference.

## Why 9-Grid

- Gemini generates all 9 cells in one context window → cross-shot consistency is naturally enforced
- Each cell can be cropped out and fed individually to the video model
- Faster than generating frames one at a time, and unlike B&W storyboards, the full-color final style prevents character contamination

## vs. Storyboard / FrameRef

| Dimension | Storyboard | FrameRef | **KeyFrames** |
|-----------|-----------|----------|---------------|
| Format | B&W line art | Full-color single frame | 3×3 full-color grid |
| Audience | Human planning | Single-shot look reference | Video model — whole-scene master |
| Focus | Camera, composition, blocking | Tone, style, lighting | Both + continuity |
| Continuity | ✅ 4-beat rhythm | ❌ Single frame | ✅ Frame-to-frame connection |
| Camera Specs | Focal length + angle | Lens + lighting + color + composition | Lens + lighting + color + composition + continuity |
| Color | None (avoid contamination) | Yes (core purpose) | Yes (lock final look) |

KeyFrames fuses storyboard's cross-shot narrative with frameRef's cinematic detail.

9 frames, 3×3 grid, read left→right top→bottom.

## Per-Frame Template

5 fields per frame, no more, no less:

```markdown
### FRAME N · SHOT_ID · Brief Description

- **Visual Content**: [What the camera sees. Visual-only description — no interpretation. Storyboard Rule 2: must be drawable, not "understandable."]
- **Camera**: [Focal length + angle + distance. E.g.: "Wide 35mm, high-angle overhead, locked-off"]
- **Lighting & Color**: [Light source direction/quality/temperature + dominant color palette + emotional function. E.g.: "Warm orange sodium streetlights scatter from above, wet asphalt reflects cold blue — 80% cool dark tones, 20% warm orange light patches"]
- **Composition & Depth**: [Foreground → Midground subject → Background. Subject placement, negative space. E.g.: "FG guardrail forms perspective leading lines → sedan centered below → BG skyline fades into fog"]
- **Continuity**: [How this frame connects to the previous/next. First frame: "Scene opener, establishes spatial tone." Last frame: "Scene closer." Middle frames: describe shot size jumps, emotional progression, eyeline direction]
```

### Field Constraints

| Field | Required | Source Fusion | Writing Rule |
|-------|----------|---------------|--------------|
| **Visual Content** | ✅ | Storyboard visual-only | 2–3 sentences. Describe only what is visible — action, position, facial detail, objects. Never write "he feels" or "this means." |
| **Camera** | ✅ | FrameRef camera specs | Focal length + angle + distance. Add movement intent if needed. |
| **Lighting & Color** | ✅ | FrameRef lighting + color | Light direction/quality/temperature + color distribution percentages across the frame |
| **Composition & Depth** | ✅ | FrameRef composition + depth | Minimum 3 planes: FG / MG subject / BG. Subject position. |
| **Continuity** | ✅ | Storyboard cross-frame narrative | Shot size jumps, emotional progression, eyeline direction, rhythm shift |

## Shot Narrative Rules (inherited from Storyboard)

1. **Frame 1 sets the baseline** — establishes composition language and spatial logic for the entire scene
2. **No adjacent frames at the same shot size** — Wide → Medium → Close → Wide alternating jumps
3. **Action is followed by reaction** — every action frame needs a reaction frame within the next 1–2 panels
4. **Eyeline direction stays consistent** — where a character looks should not arbitrarily flip between adjacent frames
5. **Lighting gradients are described, sudden shifts are flagged** — if lighting trends across frames (e.g. warm→cold), describe it in Continuity

---

## Visual Style Suffix (inherited from FrameRef)

Appended as the final block after all per-frame descriptions. Unlike storyboard's B&W suffix, keyFrames use a full-color CG anime style declaration:

**Default for 《复生协议》:**
`3D CG anime cinematic render, GANTZ:O (2016) style, Unreal Engine 5 quality, cel-shaded characters with detailed 3D face models, dark sci-fi aesthetic, Japanese CG anime film, dramatic cinematic lighting, glossy wet surfaces with raytraced reflections, 16:9 aspect ratio, high detail, cinematic quality`

> Swap per project. Always verify against `frame/style_reference.md`.

---

## Core Principles

1. **Full color, deterministic.** Unlike storyboard which deliberately avoids color — keyFrames locks color down so the video model copies it.
2. **Visual Content = purely visual.** Same strictness as Storyboard Rule 2: "eyelids relaxed, corners of mouth slightly lifted" is drawable; "he feels peaceful inside" is not.
3. **Color info written into every frame.** Light and color can shift across 9 frames (e.g. 0010 cool outdoor → 0020 warm interior) — describe per frame.
4. **Continuity is the most easily skipped field.** But it's what makes 9 frames a story, not 9 random screenshots.
5. **No `--` parameters.** Gemini reads natural language. Don't use Midjourney syntax.
